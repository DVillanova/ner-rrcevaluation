Instructions

Python 3.9
  1. Copy your Ground Truth file on the mounted folder (/gt)
  2. Implement your evaluation script. (/scripts/script.py)
  3. Add your evaluation script dependencies on (requirements.txt) file (rebuild docker)
  4. Edit the configuration
  5. If you want to show samples information, copy your samples ZIP file on the mounted folder (/gt)
  6. Verify the results page. Edit your evaluation script / configuration until looks good
Other Python version
  1. Copy your Ground Truth file on the mounted folder (/gt)
  2. Implement your evaluation script. (/scripts/script.py)
  3. Change the Docker image in the Dockerfile and add your evaluation script dependencies on (requirements.txt) file. (rebuild docker)
  4. Edit the configuration
  5. If you want to show samples information, copy your samples ZIP file on the mounted folder (/gt)
  6. Verify the results page. Edit your evaluation script / configuration until looks good
Other Languages
  1. Copy your Ground Truth file on the mounted folder (/gt)
  2. Implement the evaluation docker with you prefered language. (rebuild doker)
  3. Edit the configuration
  4. If you want to show samples information, copy your samples ZIP file on the mounted folder (/gt)
  5. Verify the results page. Edit your evaluation script / configuration until looks good
Load example

You can load an existing example: script, ground truth, samples info and configuration.

Note that if you import an example, all the files on the mounted volumes (gt, submits, scripts) will be cleared.
Files for RRC

When you have your task ready, these are the files that RRC need to set it up.

  • Ground truth
  • Python scripts (v.3.9) or your custom docker
  • Samples (optional)
  • Configuration file
  • Requirements file
  • Submition sample
Export zip

If you have customized the evaluation docker, you will have to manually export it.
